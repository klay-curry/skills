# paper-log: 论文阅读追踪

维护两篇核心文档：
- `docs/Dreamer_diffusion.md` — 研究规划（目标确认的锚点）
- `docs/self_read.md` — 论文阅读日志（本 skill 的主维护对象）

## 执行流程

### 第 0 步：确认研究目标

每次被调用时，先读取 `docs/Dreamer_diffusion.md`，向用户确认：

1. 四个研究方向是否仍然准确：
   - 🟢 方向一：RSSM 动力学 + 扩散（**主攻**）
   - 🔵 方向二：Actor 策略 + 扩散
   - 🟣 方向三：Critic 价值 + 扩散
   - ⚪ 方向四：轨迹级扩散规划
2. 是否有新增方向或调整
3. 当前阅读优先级

一句话确认即可，不用展开。如果用户说"不变"，直接进入下一步。

### 第 1 步：判断输入模式

| 模式 | 用户提供 | 触发场景 |
|------|---------|---------|
| **模式 A** | 论文 + 已有分析 markdown | 用户自己写好了分析，只需入库 |
| **模式 B** | 仅论文（URL/PDF/名称） | 需要先讲解、讨论，再生成分析 |

如果用户意图不明确，问一句："你是有写好的分析文档，还是先聊聊这篇论文？"

---

### 模式 A：直接入库

用户提供：
- 论文（arxiv URL / PDF 路径 / 论文标题）
- 已写好的分析 markdown 文件路径

执行：
1. 读取用户提供的分析 markdown
2. 读取论文（WebFetch arxiv / Read PDF / WebSearch）
3. 对照检查分析文档是否准确，有误则纠正
4. 将内容整理为 `self_read.md` 的标准条目格式，写入
5. 更新文末的「阅读统计」表格
6. 更新 `docs/Dreamer_diffusion.md` 中对应方向的论文列表（如有必要）

---

### 模式 B：讲解 → 讨论 → 生成

用户仅提供论文。这是一个多轮对话流程。

#### B.1 读取论文

- arxiv URL → `WebFetch`
- PDF → `Read`（注意 pages 限制）
- 论文名称 → `WebSearch`
- 本地文件 → `Read`

获取：标题、作者、发表 venue/年份、arxiv ID、摘要、核心方法、关键结果。

#### B.2 讲解论文（首轮输出）

用中文讲解，结构如下：

**① 一句话定位**
这篇论文解决什么问题？用非学术语言说。

**② 核心方法（费曼式）**
把核心思路讲给一个懂 ML 但不一定懂这个细分领域的人听。用类比、例子，不是翻译摘要。

**③ 关键结果**
挑最说明问题的 2-3 个数字或发现。

**④ 与你的研究方向的关系**
明确标注：这篇论文属于四个方向中的哪一个（或跨方向/基础），对你的研究有什么参考价值。

**⑤ 抛出讨论点**
结尾问用户："你对哪部分想深入了解？有没有觉得矛盾或奇怪的地方？"

#### B.3 讨论（多轮，由用户主导）

用户可能会：
- 追问方法细节
- 质疑实验设计
- 比较与其他论文的异同
- 探讨对自己研究的启发
- 要求重述某个概念

正常回应用户的问题，像讨论一样进行。不需要每轮都结构化输出。

**讨论中注意收集**：
- 用户提出的每个疑问及你的回答
- 用户表达的困惑或兴奋点
- 用户提到的"这和 XX 论文的关系"
- 用户指出的优缺点

这些将写入分析文档的「讨论记录」部分。

#### B.4 用户确认结束讨论

当用户表示"差不多了""可以记录了""就这样"，执行：

1. 生成详细分析文档 → 存入 `docs/papers/{YYYY-MM-DD}-{short-name}.md`
2. 生成精简条目 → 写入 `docs/self_read.md`
3. 更新 `self_read.md` 的「阅读统计」表格
4. 如有必要，更新 `docs/Dreamer_diffusion.md` 中对应方向的论文列表
5. 报告已完成的操作

## 详细分析文档格式

模式 B 生成的分析文档存于 `docs/papers/` 目录。使用 `references/analysis-template.md` 作为结构模板。

文件名：`{YYYY-MM-DD}-{paper-short-name}.md`

必须覆盖的章节：
1. **论文元信息** — 标题、作者、发表、arxiv ID、阅读日期
2. **一句话定位** — 解决什么问题
3. **核心方法详解** — 用自己话写的方法解释，不是翻译
4. **关键结果** — 最重要的实验发现
5. **讨论记录** — 用户提出的疑问 + 回答（逐条 Q&A 格式）
6. **优缺点分析** — 客观评估
7. **与研究方向的关系** — 属于哪个方向、贡献了什么、可借鉴什么
8. **后续行动** — check list

## self_read.md 条目格式

严格遵循现有模板。每条论文条目：

```markdown
### [论文标题]

- **作者**: 
- **发表/年份**: 
- **arXiv ID / 链接**: 
- **阅读日期**: YYYY-MM-DD

#### 核心内容（3句话以内）

（简要总结）

#### 疑问记录

1. **Q**: 问题
   - 答：回答

#### 对我研究的价值 / 佐证

- （具体价值点）

#### 后续行动

- [ ] 行动项
```

写入位置：在「记录模板」section 之前、最后一篇论文之后插入。同时更新「阅读统计」表格。

## 关联维护

### 更新 Dreamer_diffusion.md

读了一篇新论文后，检查：
- 是否属于已有四个方向之一？→ 在对应方向的论文表中添加
- 是否代表一个新方向？→ 与用户确认后添加新类别
- 是否更新某个方向的"空白分析"或"创新空间"判断？→ 与用户确认后更新

### 更新知识图谱（self_read.md 顶部 mermaid 图）

如果论文属于四个方向之一，在 mermaid 图中添加节点并连线。同时更新下方的关联表格。

## 快捷命令

用户可以使用以下简写触发特定行为：

| 用户说 | 含义 |
|--------|------|
| "记一下这篇" / "入库" | 用户已讨论完，直接进入 B.4 生成阶段 |
| "先讲论文" / "一起读" | 从 B.2 讲解开始 |
| "我写好了分析" | 模式 A |
| "更新研究方向" | 修改 Dreamer_diffusion.md 中的研究目标 |
| "看看阅读统计" | 读取 self_read.md 统计表，报告阅读进度 |
